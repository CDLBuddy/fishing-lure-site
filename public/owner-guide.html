This page summarizes how to log in, add/edit products, manage photos, and publish changes. It’s safe for you to bookmark.

1) Access & Sign-in

  1. Create a GitHub account if you don’t have one: github.com/join.
  2. Send your GitHub username to the maintainer so you can be added as a collaborator.
  3. Visit /admin.
  4. Click Login with GitHub and approve the popup. If it’s blocked, allow popups and try again.

Your edits are saved to the website’s repository and automatically deployed.

2) Products: Add / Edit / Publish

Add a product

  1. Click New Products.
  2. Fill fields:
    • ID (slug) -- short lowercase id (e.g. sb-001). Don’t change later; it becomes the URL.
    • Name and Description
    • Category -- spinnerbait / jig / crankbait / topwater / soft-plastic
    • Images -- add at least one. See image tips.
    • Variants -- for sizes/colors/weights:
      • Variant ID (e.g. v1)
      • Label (e.g. 3/8 oz • Chartreuse)
      • SKU (your internal code)
      • Stripe Price ID -- leave blank until checkout is enabled
      • Price (USD) -- optional display price for now
    • (Optional) Tags, Featured, Sort, Published At
  3. Click Save, then Publish.

Edit a product

  1. Open the product from the list.
  2. Make changes → Save → Publish.

Remove or hide a product

  • Set Status to Draft (not public) or Hidden (not listed), then Publish.
  • To retire an item permanently, switch to Hidden and coordinate full removal with the maintainer.

3) Images: Quality, Size, Filenames

  • Shape: Landscape 3:2 (our product cards expect this).
  • Size: ~1200–1600px wide.
  • Format: JPG/JPEG at quality ~80–85.
  • Filenames: Lowercase, hyphens, no spaces (river-shimmer.jpg). Case matters on hosting.

If a new image doesn’t appear right away, hard-refresh. For instant change, upload with a new filename.

4) Statuses: Draft vs Active vs Hidden

  • Draft -- Not live. Safe to work in.
  • Active -- Live on the site (catalog/product pages/sitemap).
  • Hidden -- Not listed; kept out of sitemap.

5) Variants (sizes, colors, weights)

Use variants for different sizes, colors, weights, or hooks of the same lure.

  • Give each variant a stable Variant ID (v1, v2).
  • Label is what shoppers see (e.g. 1/2 oz • Brown/Orange).
  • SKU is your internal code.
  • Stripe Price ID stays blank until checkout is enabled.

6) Featured Products

Toggle Featured to highlight items on the homepage/featured sections (if configured). Use sparingly.

7) Cart & Checkout (Stripe)

  • The site supports a shopping cart now.
  • Checkout is currently disabled until Stripe is connected.
  • When ready:
    • Configure products/prices in Stripe.
    • Paste Stripe price_... IDs into each variant.
    • We’ll enable checkout and test in Stripe’s Test Mode.

8) What Happens When You Publish

  1. The admin saves your edits to the repository.
  2. The hosting platform rebuilds and deploys automatically.
  3. When the deploy finishes, refresh the site to see changes.

If you don’t see the change immediately, give it a moment and refresh again.

9) Troubleshooting

Login loop or "Authorization failed"
  • Allow popups for the site.
  • Verify you’re logged into the invited GitHub account.
  • Try a private/incognito window.
Change not visible
  • Did you click Publish (not only Save)?
  • Wait for the deploy to finish; refresh the page.
  • Confirm Status is Active.
Image not updating
  • Hard refresh the page.
  • Upload with a new filename for instant change.
"Not found" after renaming
  • Don’t change the product ID (slug) after publishing.
  • If it must change, coordinate a redirect with the maintainer.

10) FAQ

Can I create new categories?
Yes, but keep to the approved list for now: spinnerbait, jig, crankbait, topwater, soft-plastic. More can be added by the maintainer.

Can I reorder products?
Yes--use the Sort field (lower numbers appear first). If you don’t see changes, ask the maintainer to enable sorting on the catalog.

Can I upload PNG or HEIC?
JPG is recommended for best size/quality. Convert HEIC to JPG before upload.

Can I run sales or discounts?
Once Stripe is live, we can use sale prices or coupon codes and reflect that on the site.
